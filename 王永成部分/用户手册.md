## 1 引言

### 1.1 编写目的

本手册面向**普通浏览用户**和**信息分析员**，说明如何登录系统、浏览热点趋势、检索信息明细、查看详情以及导出分析报表。读者无需了解后端实现细节，按本手册步骤即可完成日常查询与分析操作。

### 1.2 适用对象

| 用户角色 | 主要用途 | 本手册相关章节 |
|----------|----------|----------------|
| 普通浏览用户 | 查看热点榜、搜索信息、阅读详情 | 第 3、4、5 章 |
| 信息分析员 | 趋势研判、主题对比、导出报表 | 第 4、5、6 章 |

系统管理员的操作（数据源配置、采集任务、审计日志等）请参阅《操作手册》。

### 1.3 术语说明

| 术语 | 说明 |
|------|------|
| 热点榜 | 按热度分数排序的主题列表 |
| 趋势方向 | up（上升）、down（下降）、stable（稳定）、surge（突增） |
| 质量评分 | 清洗模块对单条信息完整度的 0~100 分评价 |
| 信息明细 | 经采集和清洗后的结构化信息条目 |

### 1.4 运行环境

**表1-1 客户端环境要求**

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Edge 90+、Firefox 88+ |
| 屏幕分辨率 | 建议 1280×720 及以上 |
| 网络 | 能访问系统服务器（默认 `http://127.0.0.1:8000`） |

## 2 系统访问

### 2.1 启动与访问

1. 确认后端服务已启动（由管理员或开发人员执行）：

```powershell
python -m app.main --host 127.0.0.1 --port 8000
```

2. 在浏览器地址栏输入系统地址，进入**系统首页（Dashboard）**。
3. 首次访问时，系统自动加载演示数据，首页应显示热点主题和最新信息摘要。

### 2.2 页面导航

**表2-1 主导航菜单**

| 菜单项 | 页面 | 功能 |
|--------|------|------|
| 首页 | Dashboard | 今日热点、趋势概览、来源分布 |
| 趋势分析 | /trends | 主题榜单、趋势方向、主题详情 |
| 信息搜索 | /items | 关键词检索、筛选、详情查看 |
| 报表中心 | /reports | 导出分析报表（分析员） |

### 2.3 健康检查

若页面无法加载数据，可在浏览器访问：

```text
http://127.0.0.1:8000/api/v1/health
```

正常时应返回 `"status": "ok"`。若返回错误，请联系系统管理员。

## 3 首页使用说明

### 3.1 页面布局

首页分为四个区域：

1. **顶部导航栏**：系统名称、主导航、当前用户（如有登录）。
2. **今日热点榜**：展示热度排名前 10 的主题及分数、趋势方向。
3. **最新信息摘要**：展示最近入库的信息标题、来源、发布时间。
4. **来源分布概览**：按 news / forum / api / csv 统计数据条数占比。

### 3.2 查看今日热点

**操作步骤：**

1. 进入首页，默认展示「今日热点榜」。
2. 每条热点显示：**主题名称**、**热度分数**、**趋势方向图标**。
3. 点击某一主题行，跳转至**趋势分析页**并自动选中该主题。
4. 点击「刷新」按钮重新加载最新数据。

**表3-1 趋势方向图标含义**

| 方向 | 图标 | 含义 |
|------|------|------|
| up | ↑ | 热度较上一周期上升超过 20% |
| down | ↓ | 热度下降超过 20% |
| stable | → | 热度变化在 ±20% 以内 |
| surge | ⚡ | 新出现或突增主题 |

### 3.3 浏览最新信息

1. 在首页「最新信息」区域查看最近 5~10 条信息。
2. 每条显示标题、来源名称、发布时间、质量评分。
3. 点击标题进入**信息详情页**，查看完整正文、关键词和关联主题。

## 4 趋势分析

### 4.1 进入趋势分析页

- 方式一：点击顶部导航「趋势分析」。
- 方式二：在首页热点榜中点击某一主题。

### 4.2 查看趋势榜单

**表4-1 趋势榜单字段说明**

| 列名 | 说明 |
|------|------|
| 排名 | 按 score 降序排列 |
| 主题 | 关键词或主题名称 |
| 热度分数 | 综合频次与时间衰减的计算结果 |
| 趋势方向 | up / down / stable / surge |
| 统计周期 | period_start ~ period_end |

**操作步骤：**

1. 打开趋势分析页，默认展示前 20 个主题。
2. 使用「显示条数」下拉框选择 10 / 20 / 50。
3. 点击表头「热度分数」可切换升序/降序（界面实现后可用）。
4. 点击某一主题行的「查看详情」，展开该主题的来源信息列表。

### 4.3 查看单主题详情

选中主题后，右侧或下方展示：

- 主题名称与当前热度分数
- 趋势方向及简要解读
- 统计周期（起止日期）
- 相关信息列表（标题、来源、发布时间链接）

**对应 API（供联调参考）：**

```text
GET /api/v1/trends?limit=20
GET /api/v1/trends/{topic}
```

**响应示例：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "topics": [
      {
        "topic": "人工智能",
        "score": 24.0,
        "direction": "up",
        "period_start": "2026-06-15",
        "period_end": "2026-06-18"
      }
    ]
  }
}
```

### 4.4 趋势研判建议

分析员在使用趋势榜时，建议关注：

1. **surge** 方向主题：可能是新兴热点，需结合信息明细核实。
2. **score 高且 direction 为 up**：持续升温主题，适合纳入报告。
3. **direction 为 down 但 score 仍高**：存量热点降温，可对比历史周期。

## 5 信息搜索

### 5.1 进入搜索页

点击顶部导航「信息搜索」，进入信息检索页面。

### 5.2 基本搜索

**操作步骤：**

1. 在搜索框输入关键词，如 `人工智能`、`新能源汽车`。
2. 点击「搜索」或按 Enter 键。
3. 结果列表展示匹配的信息条目。

**表5-1 搜索结果字段**

| 字段 | 说明 |
|------|------|
| 标题 | normalized_title 或原始 title |
| 来源 | 数据源名称（source_id 对应） |
| 发布时间 | published_at，ISO8601 格式 |
| 关键词 | 清洗模块提取的关键词，逗号分隔 |
| 质量评分 | 0~100，越高表示内容越完整 |

### 5.3 高级筛选

在「高级筛选」面板中可组合以下条件：

| 筛选项 | 说明 |
|--------|------|
| 关键词 | 标题、正文、关键词模糊匹配 |
| 来源 | 按 data_sources 下拉选择 |
| 显示条数 | 默认 50，最大 100 |

**对应 API：**

```text
GET /api/v1/items?keyword=人工智能&limit=50
GET /api/v1/items?keyword=人工智能&source_id=1&limit=20
```

### 5.4 查看信息详情

1. 在搜索结果中点击某条信息的「详情」链接。
2. 详情页展示：

**表5-2 详情页内容区块**

| 区块 | 内容 |
|------|------|
| 基本信息 | 标题、来源、作者、发布时间、原文链接 |
| 正文 | 清洗后的 normalized_content |
| 关键词 | 系统提取的关键词标签 |
| 质量评分 | 数值及简要说明（如「正文充实」「链接有效」） |
| 关联主题 | 与该信息相关的热点主题（如有） |

**对应 API：**

```text
GET /api/v1/items/{item_id}
```

3. 点击「返回列表」回到搜索结果页，保留上次筛选条件。

## 6 报表导出

### 6.1 适用角色

报表导出功能主要面向**信息分析员**。普通浏览用户若需导出，请联系分析员或管理员开通权限。

### 6.2 导出入口

1. 进入「报表中心」页面，或
2. 在趋势分析页点击「导出当前榜单」，或
3. 在信息搜索页勾选条目后点击「导出选中」。

### 6.3 导出步骤

1. 选择报表类型：
   - **摘要报表（summary）**：热点主题 + 统计周期概览
   - **明细报表（detail）**：信息条目列表（界面扩展项）
2. 选择导出格式：当前版本支持 summary 类型记录；Excel/PDF 导出为前端展示层扩展功能。
3. 填写「生成人」标识（默认当前用户名）。
4. 点击「生成报表」，系统返回报表记录。

**对应 API：**

```text
POST /api/v1/reports
Content-Type: application/json

{
  "format": "summary",
  "generated_by": "分析员张三"
}
```

**成功响应：**

```json
{
  "code": 0,
  "data": {
    "id": 1,
    "report_name": "trend-report-2026-06-23T12:00:00+00:00.summary",
    "report_type": "summary",
    "file_path": "reports/trend-report-....summary",
    "generated_by": "分析员张三",
    "generated_at": "2026-06-23T12:00:00+00:00"
  }
}
```

5. 在报表中心查看历史导出记录，点击「下载」获取文件（路径以 file_path 为准）。

### 6.4 报表使用建议

- 答辩或汇报前，建议先导出一版 summary 报表作为附件。
- 导出前确认趋势数据已更新（管理员需先完成采集任务）。
- 报表文件名含时间戳，便于版本区分。

## 7 常见问题

**表7-1 常见问题与处理**

| 问题 | 可能原因 | 处理办法 |
|------|----------|----------|
| 首页无热点数据 | 尚未执行采集任务 | 联系管理员启动采集 |
| 搜索无结果 | 关键词不匹配或数据未入库 | 更换关键词或放宽筛选 |
| 趋势榜为空 | topic_trends 表无记录 | 确认采集成功后刷新页面 |
| 详情页链接打不开 | 原文 URL 为演示地址 | 属正常现象，以系统内正文为准 |
| 报表生成失败 | 服务异常或权限不足 | 查看 health 接口，联系管理员 |
| 页面加载超过 3 秒 | 数据量过大或网络延迟 | 缩小 limit，检查服务器负载 |
