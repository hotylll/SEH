## 1 设计概述

### 1.1 设计目标

本说明定义信息收集整合系统 Web 前端的页面结构、视觉规范、交互流程和组件布局，覆盖**系统首页**、**趋势分析页**、**信息搜索页**及管理员相关页面。设计遵循简洁、信息密度适中、适合答辩演示的原则，并与后端 REST API 字段一一对应。

### 1.2 设计依据

- 《软件需求说明书》REQ-014、REQ-015、REQ-016
- 《概要设计说明书》展示层与接口层划分
- 现有 API：`/api/v1/trends`、`/api/v1/items`、`/api/v1/sources` 等

### 1.3 技术建议

| 项目 | 建议方案 |
|------|----------|
| 框架 | 原生 HTML/CSS/JavaScript |
| 图表 | 原生 DOM 条形图 |
| UI 组件库 | 暂不引入，保持课程演示部署简单 |
| 布局 | 响应式，最小宽度 1280px |

### 4.3 区块说明

**表4-1 首页区块与 API 映射**

| 区块 | 数据来源 | API |
|------|----------|-----|
| 热点主题数 | trends 列表长度 | GET /api/v1/trends?limit=20 |
| 信息总数 | items total | GET /api/v1/items?limit=1 |
| 数据源数 | sources 列表 | GET /api/v1/sources |
| 今日热点榜 | topics 数组 | GET /api/v1/trends?limit=10 |
| 来源分布 | 按 source_id 聚合 items | GET /api/v1/items + 前端聚合 |
| 最新信息 | items 前 5 条 | GET /api/v1/items?limit=5 |

### 4.4 交互说明

1. 页面加载时并行请求 trends、items、sources、health。
2. 点击热点榜「详情」→ 跳转 `/trends?topic=人工智能`。
3. 点击最新信息标题 → 跳转 `/items/{id}`。
4. 「刷新」重新拉取全部接口；「导出摘要」调用 POST /api/v1/reports。

3. 右侧详情区展示 series 数据及关联 items（通过 keyword=topic 搜索 items）。
4. 空状态：显示「暂无趋势数据，请联系管理员执行采集任务」。

## 9 响应式与无障碍

### 9.1 断点

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| Desktop | ≥1280px | 完整双栏布局 |
| Tablet | 768~1279px | 趋势详情移至下方 |
| Mobile | <768px | 导航收折为汉堡菜单，表格横向滚动 |

### 9.2 无障碍

- 趋势方向除颜色外必须带图标或文字标签。
- 表单控件需关联 label。
- 主按钮与背景对比度 ≥ 4.5:1。
