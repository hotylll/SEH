## 1 测试目标

本测试计划用于规范信息收集整合系统的测试范围、测试方法、测试环境和通过标准。测试工作覆盖数据源配置、采集任务、数据清洗、趋势分析、接口调用、前端演示和性能冒烟验证，最终由组长统一审查并纳入项目归档。

## 4 测试策略

1. 先执行单元测试，确认清洗、评分、趋势计算等基础函数稳定。
2. 再执行存储层测试，确认数据库写入、去重、任务状态和数据源状态逻辑正确。
3. 随后执行接口集成测试，确认 API 能完成创建数据源、采集、检索、趋势查询的闭环。
4. 最后执行性能冒烟测试，确认 10000 条演示数据下的检索响应时间满足课程演示要求。
5. 每次成员代码合并后，由组长运行全量测试并检查前端演示流程。

## 5 通过标准

| 编号 | 标准 | 判定 |
|:----:|------|:----:|
| P-001 | 全部自动化测试通过 | 必须 |
| P-002 | 健康检查接口返回正常状态 | 必须 |
| P-003 | 数据源支持 `news/forum/api/csv` 类型 | 必须 |
| P-004 | 调度字段支持 `manual/hourly/daily/weekly` | 必须 |
| P-005 | 禁用数据源不能执行采集任务 | 必须 |
| P-006 | 重复采集不会重复写入清洗数据 | 必须 |
| P-007 | 趋势方向支持五类状态 | 必须 |
| P-008 | 10000 条数据检索响应小于 3 秒 | 必须 |
| P-009 | 前端页面能展示首页、趋势、检索、数据源、报表 | 必须 |

## 6 风险与处理

| 风险 | 影响 | 处理方式 |
|------|------|----------|
| 成员模块接口字段不一致 | 影响联调 | 由组长统一接口字段和数据结构 |
| 前端连接不到后端 | 影响演示 | 检查后端端口与前端 API 地址 |
| 重复数据导致采集数量为 0 | 容易误判失败 | 说明去重机制，必要时新增关键词数据源 |
| 二进制文档无法自动检查 | 影响最终归档 | 由组长人工打开复核 |
