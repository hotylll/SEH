## 2 推荐演示流程

### 2.1 系统首页

1. 打开 `frontend/index.html`。
2. 确认服务状态显示 `ok`。
3. 展示热点主题数、信息总数、数据源数。
4. 展示今日热点榜和热度分布。

截图建议：系统首页全屏截图。

### 2.2 数据源管理

1. 切换到“数据源管理”。
2. 新增一个数据源：

```text
名称：答辩演示源
类型：news
endpoint：https://example.com/demo
keywords：人工智能,新能源汽车
schedule：weekly
status：enabled
```

3. 点击“采集”按钮。
4. 确认提示中出现任务状态和新增条数。

截图建议：数据源列表 + 采集结果提示。

### 2.3 信息搜索

1. 切换到“信息搜索”。
2. 输入关键词 `人工智能`。
3. 点击搜索。
4. 展示标题、来源、发布时间、关键词、质量评分。

截图建议：搜索结果表格。

### 2.4 趋势分析

1. 切换到“趋势分析”。
2. 点击刷新趋势。
3. 展示主题、分数、趋势方向和统计周期。
4. 说明趋势方向支持 `surge/up/stable/down/plunge` 五类。

截图建议：趋势榜单。

### 2.5 报表中心

1. 切换到“报表中心”。
2. 选择 `summary`。
3. 生成人填写 `组长`。
4. 点击生成报表。
5. 展示报表名称和文件路径。

截图建议：报表生成结果。

## 4 答辩讲解要点

1. 组长负责整体架构、核心代码、接口联调、测试验收和最终归档。
2. 系统采用 Python 标准库实现，降低部署复杂度，适合课程演示和源码说明。
3. 数据流为：数据源配置 → 采集任务 → 原始数据入库 → 清洗评分 → 趋势分析 → 前端展示。
4. SQLite 表结构覆盖用户、数据源、任务、原始数据、清洗数据、趋势、报表、审计日志。
5. 测试覆盖单元测试、接口集成测试和 10000 条数据性能测试。
6. 后续扩展方向包括真实爬虫接入、登录权限、定时调度、Excel/PDF 文件导出。

## 5 演示风险与处理

| 风险 | 处理方式 |
|------|----------|
| 8000 端口占用 | 改用 `--port 8001`，并在前端 API 输入框修改地址 |
| 页面无数据 | 点击数据源管理中的“采集”，再刷新首页 |
| 浏览器跨域错误 | 确认后端已启动并使用当前版本 `app/api.py` |
| 重复采集新增条数为 0 | 属于去重机制正常现象，可新增不同关键词数据源重新采集 |
| 报表只有记录无真实文件 | 当前版本为报表记录功能，Excel/PDF 为后续扩展 |
