## 第 4 页 系统总体架构

- 表现层：前端演示页、Dashboard、趋势分析、信息搜索、报表中心。
- 接口层：统一 REST API，提供健康检查、数据源、采集、检索、趋势、报表接口。
- 业务层：采集任务、清洗评分、趋势计算、报表记录。
- 数据层：SQLite 持久化，包含 users、data_sources、collect_tasks、raw_items、clean_items、topic_trends、reports、audit_logs。

## 第 5 页 核心业务流程

- 配置数据源：支持 news / forum / api / csv 四类来源。
- 启动采集任务：按来源类型和关键词生成演示数据。
- 清洗处理：HTML 清理、字段规范化、关键词提取、质量评分。
- 趋势分析：按关键词统计热度，支持时间衰减和五类趋势方向。
- 前端展示：热点榜、热度分布、信息检索、数据源管理、报表生成。

## 第 6 页 关键技术实现

- 技术栈保持轻量：Python 3 标准库 + SQLite + 静态前端页面。
- 后端服务：ThreadingHTTPServer + BaseHTTPRequestHandler。
- 数据访问：Repository 封装 SQLite 会话、事务和表结构。
- 质量评分：标题长度、正文长度、URL、关键词密度、乱码检测、字段完整性等维度。
- 趋势算法：surge / up / stable / down / plunge 五态方向 + 0.90^days 时间衰减。

## 第 7 页 接口与数据库设计

- 主要接口：/health、/sources、/tasks/collect、/items、/trends、/reports。
- 统一响应格式：code、message、data、trace_id。
- 数据库共 8 张核心表，覆盖配置、任务、原始数据、清洗数据、趋势、报表、审计日志。
- raw_hash 唯一约束用于数据去重。
- audit_logs 记录关键操作，体现基本治理能力。

## 第 8 页 前端可视化演示

- frontend/index.html 为课程答辩演示入口。
