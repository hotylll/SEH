## 1 项目概述

### 1.1 项目名称

信息收集整合系统（暂定名，后续可修改）

### 1.2 项目定位

参考 **BettaFish**（多智能体舆情分析）与 **TrendRadar**（热点趋势雷达），构建一个多源信息采集 → 清洗处理 → 整合分析 → 可视化展示的一站式平台。

### 1.3 核心功能

- 多源信息采集引擎
- 数据清洗与归并模块
- 信息存储与索引模块
- 趋势分析与洞察模块
- 可视化展示与报表模块
- 用户管理与配置模块

### 1.4 标准依据

本项目的全部文档遵循 **GB856T——88** 和 **GB8567——88** 国家标准。

#### 2. 罗昆昊：清洗与存储负责人

### 2.5 当前协作原则

1. 各成员先按自己的模块把“文档初稿 + 功能说明”补齐，再统一交给组长审查。
2. 文档与代码同步推进，模块做完后要立刻补对应的说明、截图和测试记录。
3. 每个人交付时必须注明自己负责的部分，便于后续汇总到总文档和答辩材料里。
4. 组长负责统一风格、检查格式、合并内容和最后验收，不再重复分散细节工作。

### 4.2 代码要求

1. **语言标准**：Python 3.10+，遵循 PEP8 规范
2. **模块化**：每个功能独立成模块，降低耦合
3. **注释规范**：关键函数/类须有 docstring，复杂逻辑须有行注释
4. **Git 提交**：提交信息格式 `[模块名] 描述`，如 `[采集] 新增微博爬虫`
5. **代码评审**：关键模块合并前须经过组长审查

### 4.3 协作要求

1. **每周例会**：每周一晚进行进度同步，所有人汇报本周完成/下周计划
2. **通讯渠道**：建立微信群，日常沟通在群内进行
3. **问题上报**：遇到技术卡点超过1天无法解决，立即上报组长协调
4. **禁止拖延**：除非特殊情况，每个阶段任务不得延迟超过3天
5. **会议纪要**：每次例会由组长记录，发到群里

### 4.4 其他要求

1. 所有文档和代码最终版权归小组共有
2. 各组员须保管好自己的工作副本
3. 参考外部代码须注明出处
4. 不得抄袭，须有自己的设计和实现
